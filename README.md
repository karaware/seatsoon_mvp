# SeatSoon

混雑したフードコートで、席を探している人ともうすぐ席を空ける人をゆるくつなぐスマホWebアプリMVPです。

## セットアップ

```bash
npm install
cp .env.example .env.local
```

`.env.local` にSupabaseの値を設定します。

```env
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
```

Supabase SQL Editorで `supabase/schema.sql` を実行してください。

既にテーブルを作成済みの場合も、Policyを更新するため同じSQLを再実行できます。

## 開発

```bash
npm run dev
```

## MVP仕様

- ログイン不要。ブラウザの `localStorage` に匿名ユーザーIDを保存します。
- スマホ利用を優先し、施設別URL `/f/[slug]` で有効な投稿を先に表示します。
- 投稿画面は `/f/[slug]/new?type=offer` と `/f/[slug]/new?type=request` で投稿種別を固定します。
- 一覧は「席探してます」「席譲ります」の件数サマリーとタブで切り替えます。
- 予定時間はUIから外しています。
- 投稿は10分で期限切れ扱いになり、一覧から消えます。
- 投稿者本人だけが投稿を3分延長または終了できます。
- 同じブラウザの匿名ユーザーは、同じ施設で有効な投稿を複数作れません。
- 「席譲ります」の詳細から「この席に向かう」を記録できます。
- 向かっている本人が「席を確保できた」を押すと、マッチ完了として記録し、対象投稿を終了します。
- 一覧は15秒ごとに自動更新されます。

## MVPで見る指標

- 施設別・時間帯別の有効投稿数
- 「席探してます」と「席譲ります」の投稿比率
- 「この席に向かう」が押された件数と、席譲り投稿に対する反応率
- 「席を確保できた」が押された件数と、マッチ完了率
- 投稿作成から「この席に向かう」までの時間
- 「この席に向かう」から「席を確保できた」までの時間
- 重複投稿がブロックされた件数
